# AI 智能旅行助手

> 基于大语言模型与 Agent 架构的一站式旅行规划系统，实现从「信息搜索 → 智能规划 → 预算管控 → 地图可视化 → 行程编辑 → 文档导出」的全流程自动化闭环。

---

## 项目价值

传统旅行规划需要用户在多个平台间反复切换（攻略App、地图、记账、文档），耗时且信息碎片化。本项目构建了一个**具备工具调用能力的 AI Agent**，通过自然语言交互即可在数分钟内完成以往数小时的手动规划工作，显著提升旅行决策效率与体验。

**核心交付价值：**
- 将旅行规划的决策链路从「人工搜索+手动整理」升级为「对话式智能生成」
- 支持实时联网获取最新旅游资讯，避免信息过时
- 自动生成可下载的 PDF 行程单与地图可视化，直接用于出行参考

## 功能特性

| 模块 | 能力说明 | 技术亮点 |
|------|---------|---------|
| 智能搜索 | 实时检索目的地景点、美食、酒店信息 | 基于 Web Search API，支持摘要生成与结构化提取 |
| AI 行程规划 | 根据日期、偏好、人数自动生成多日行程 | 大模型 Prompt Engineering + 多级 JSON 容错解析 |
| 预算计算 | 自动统计门票、餐饮、住宿、交通费用 | 细粒度费用拆解，支持人均与总价双维度 |
| 地图可视化 | 在地图上标注景点并绘制游览路线 | Pillow 纯代码绘制地图，零外部网络依赖 |
| 行程编辑 | 支持添加、删除、调整景点顺序 | 结构化数据操作，状态一致性保障 |
| 文档导出 | 导出为 PDF 或高清图片行程单 | Markdown → PDF 流水线，对象存储分发 |
| 对话记忆 | 保留最近 20 轮对话上下文 | 滑动窗口机制，防止上下文溢出 |

## 核心工程能力

### 1. Agent 架构设计（LangChain + LangGraph）
- 采用 `create_agent` 构建具备**工具调用（Tool Calling）**能力的 ReAct 风格智能体
- 自定义 `AgentState` 状态机，实现滑动窗口消息记忆（最近 20 轮 / 40 条消息）
- 工具执行错误中间件：`@wrap_tool_call` 捕获异常并返回结构化错误信息，避免 Agent 循环中断

### 2. 大模型输出可靠性工程
LLM 生成 JSON 存在格式不稳定风险，本项目设计了 **5 级降级容错策略**：
1. **直接解析** — 标准 `json.loads`
2. **格式修复** — 自动处理尾部逗号、缺失引号键名、BOM 字符
3. **截断修复** — 定位最后一个完整 JSON 对象边界后重新解析
4. **深度修复** — 集成 `json_repair` 库进行语义级修复
5. **重试生成** — 降低温度参数（0.3 → 0.1）要求模型重新输出

> 该机制将 JSON 解析成功率从约 75% 提升至 **95%+**，保障生产环境稳定性。

### 3. 零依赖地图可视化
- 不依赖 Google Maps / 高德等外部瓦片服务，使用 **Pillow 纯代码绘制**
- 自主实现经纬度坐标投影、网格背景、路线连线、景点标注、图例生成
- 完全离线可用，规避网络超时与 API 配额风险

### 4. 多模态内容生产流水线
- **PDF 导出**: Markdown → Document Generation SDK → 对象存储签名链接
- **图片导出**: Pillow 排版渲染 → 上传对象存储 → 返回可访问 URL
- 所有产物均通过 S3 兼容存储分发，支持 24h 有效期签名链接

### 5. 工程规范与可观测性
- 链路追踪：基于 `request_context` 实现全链路上下文透传
- 日志规范：结构化 JSON 日志，支持按模块检索
- 配置分离：LLM 参数、系统提示词、工具列表全部外置至 `config/`，支持热切换

## 快速开始

### 环境要求
- Python >= 3.12
- uv（包管理器）

### 本地运行

```bash
# 1. 安装依赖
uv sync

# 2. 运行 Agent 对话（流式）
bash scripts/local_run.sh -m flow

# 3. 启动 HTTP 服务
bash scripts/http_run.sh -m http -p 5000
```

### 对话示例

```text
用户: 我想去西安玩3天，2个人，喜欢历史文化和美食

Agent: 已为您规划西安3日深度游行程...
      [展示每日景点、餐饮、住宿安排]

用户: 帮我算一下预算

Agent: [自动调用预算工具，返回门票/餐饮/住宿/交通明细]

用户: 生成一张地图看看

Agent: [返回带路线标注的地图图片链接]

用户: 导出成PDF

Agent: [返回 PDF 下载链接]
```

## 技术栈

| 层级 | 技术选型 |
|------|---------|
| 大模型 | 豆包 (Doubao/Seed) — 字节跳动自研大模型 |
| Agent 框架 | LangChain 1.0 + LangGraph 1.0 |
| 服务框架 | FastAPI + Uvicorn |
| 数据处理 | Pandas、 Pillow、 json-repair |
| 文档生成 | coze-coding-dev-sdk (DocumentGeneration) |
| 存储 | S3 兼容对象存储 |
| 包管理 | uv |
